C#

1 Введение в C#
1.1 Язык C# и платформа .NET

Преимущества платформы

  • Поддержка нескольких языков (С#, VB.NET, C++, F#)
  • Кроссплатформенность
  • Мощная библиотека классов
  • Разнообразие технологий
  • Высокая производительность
  • Автоматическое управление памятью
  • Безопасность кода

Управляемый и неуправляемый код

Управляемый код:

  • Управляемый код (Managed code): приложение управляется средой(Common Language Runtime(CLR) для C#), загружающей приложение и управляющей памятью

Неуправляемый код:

  • Неуправляемый код (Unmanaged code): приложение компилируется в машинный код и не управляется никакой средой, памятью управляет разработчик

JIT- и AOT-компиляция

JIT-компиляция (Just-In-Time: точно в срок):

  • Компиляция происходит только части, к которой идет непосредственное обращение
  • Скомпилированная часть приложения сохраняется до завершения работы программы

AOT-компиляция (Ahead-of-Time: заранее):

  • Перед выполнением приложение компилируется в машинный код определенной платформы
  • Программа становится непереносимой
1.2 Работа с .NET CLI

Основные команды .NET CLI:

  • Создание нового проекта/файла на основе шаблона:
  • Console
    C:\Users\User> dotnet new шаблон Название
  • Добавление зависимостей к проекту:
  • Console
    C:\Users\User> dotnet add package Название/Ссылка/Путь
  • Востановление зависимстей, указанных в проекте:
  • Console
    C:\Users\User> dotnet restore Название/Ссылка/Путь
  • Сборка проекта и его зависимостей:
  • Console
    C:\Users\User> dotnet build
  • Запуск исходного проекта без явной сборки:
  • Console
    C:\Users\User> dotnet run
  • Упаковка приложения без развертывания:
  • Console
    C:\Users\User> dotnet publish
  • Очистка входных данных сборки (папки bin и obj):
  • Console
    C:\Users\User> dotnet clean
  • Запуск тестов:
  • Console
    C:\Users\User> dotnet test
  • Запуск команды (обычно run или test) и перезапуск при изменении файлов:
  • Console
    C:\Users\User> dotnet watch команда/проект

Основные флаги команд:

  • Определение конфигурации сборки проекта:
  • Console
    C:\Users\User> --configuration конфигурация
    C:\Users\User> -c конфигурация
  • Определение директории для размещения выходных файлов:
  • Console
    C:\Users\User> --output ИмяПапки
    C:\Users\User> -o ИмяПапки
  • Определение целевого фреймворка:
  • Console
    C:\Users\User> --framework версия
    C:\Users\User> -f версия
  • Уровень детализации логов:
  • Console
    C:\Users\User> --verbosity уровень
    C:\Users\User> -v уровень
  • Пропуск этапа восстановления зависимостей при сборке/запуске:
  • Console
    C:\Users\User> --no-restore

Виды конфигурации проекта:

  • Debug (по умолчанию): версия для разработки (без оптимизаций, с отладочными данными)
  • Release: версия для публикации (максимальная оптимизация, без лишних данных)

Логи

Логи: структурированные записи о событиях (действия/ошибки/предупреждения) в работе приложения, используемые для отладки и мониторинга

Уровни детализации логов:

  • q/quiet: только итоговое резюме сборки (успех/провал) и критические ошибки, из-за которых процесс прервался
  • m/minimal: ошибки, предупреждения, сообщения высокой важности и сводку по проекту
  • n/normal (по умолчанию): список всех обрабатываемых проектов, основные этапы (targets) и статус их выполнения.
  • d/detailed: подробности выполнения каждой задачи внутри этапов
  • diag/diagnostic: абсолютно всё, значения всех переменных MSBuild, полные пути, переменные окружения, детальные замеры времени на каждую задачу
2 Основы программирования на C#
2.1 Структура программы

Директива подключения (Directives):

  • using: директива, импортирующая внешние файлы
  • Упрощает разрешение имен (Name Resolution), избавляя от необходимости использовать полные уточненные имена (Fully Qualified Names)
  • Подключение объекта:
  • Code
    using ИмяОбъекта;

Модификаторы using:

  • using static: импортирует статические члены класса
  • global using: объявляет зависимости на уровне всего проекта

Пространство имен (namespace):

  • namespace: логический контейнер, используемый для группировки связанных типов (классов, интерфейсов, структур) и предотвращения конфликтов имен
  • Пространства имен могут иметь древовидную структуру
  • Определение пространства имен:
  • Code
    namespace ИмяПространстваИмен

Объявление типа (Type Declaration):

  • Тип: фундаментальный шаблон определяющий структуру и поведение объекта
  • Члены типа: включают в себя поля (состояние), методы (поведение), свойства (инкапсулированный доступ), события и конструкторы, место хранения (стек или куча)
  • Модификаторы: определяют уровень доступа (public, private, internal) и характеристики поведения (static, sealed, abstract)
  • Определение типа:
  • Code
    модификаторы тип ИмяТипа
    {                       
       // Члены типа        
    }

Точка входа (Entry Point):

  • Метод Main: точка входа, с которого исполняющая среда CLR начинает выполнение потока управления
  • Сигнатура: должен быть объявлен как static
  • Параметры: принимает массив строк string[] args, представляющий аргументы командной строки, переданные при запуске процесса
  • Возвращает int (код завершения) или void
  • Поддерживает асинхронность (static async Task)
  • Объявление точки входа:
  • Code
    static void Main(string[] args)
    {                              
       // Инструкции               
    }

Инструкции и операторы (Statements & Operators):

  • Инструкция: базовая, завершенная команда, выполняющая действие
  • Оператор: специальный символ или ключевое слово, выполняющее операции над одним или несколькими операндами
  • Каждая инструкция завершается символом ;
  • Группировка инструкций осуществляется с помощью фигурных скобок { }, образуя блок кода (Code Block)
  • Инструкции выполняются последовательно (за исключением управляющих конструкций if, switch, for, while) в контексте стека вызовов текущего потока
  • Code
    Инструкция;

Комментарии:

  • Применяются для пояснения кода, повышения его читаемости и помощи в документировании
  • При компиляции комментарии игнорируются и не оказывают никакого влияние на работу приложения и на его размер
  • Code
    //Однострочный комментарий
    /*Многострочный           
      комментарий*/

Структура проекта:

Решение (Solution - .sln):

  • Объединяет один или несколько связанных проектов
  • Управляет группой проектов как единым целым

Проект (Project - .csproj): основная единица сборки (проект компилируется в выходной файл: исполняемый файл (.exe) или библиотеку (.dll)):

  • Целевая версия платформы
  • Зависимости (NuGet-пакеты и другие библиотеки)
  • Настройки компиляции

Зависимости (Dependencies): логический раздел для внешних связей:

  • Frameworks: базовые библиотеки .NET
  • Packages: сторонние библиотеки, загруженные через менеджер пакетов NuGet
  • Projects: ссылки на другие проекты в рамках того же решения

Программные файлы (.cs): файлы с исходным кодом:

  • Program.cs — файл, содержащий точку входа в приложение
  • Папки(namespace) с классами, интерфейсами и перечислениями

Физические папки (Служебные):

  • obj/: Содержит временные объектные файлы, которые нужны компилятору в процессе сборки
  • bin/: Содержат итоговый результат компиляции (бинарные файлы):
    • Debug: для разработки
    • Release: оптимизированная версия

Иерархия проекта:

  • Console
    ИмяРешения.sln                  // Решение                      
    │                                                               
    ├── ИмяПроекта.App/             // Основной проект (Console App)
    │   ├── ИмяПроекта.App.csproj   // Настройки проекта            
    │   ├── Program.cs              // Точка входа                  
    │   ├── ИмяТипа1.cs             // Файл с описанием класса      
    │   └── ИмяПапки/               // Папка (подпространство имен) 
    │       └── ИмяТипа2.cs                                         
    │                                                               
    └── ИмяПроекта.Tests/           // Проект с модульными тестами  
        ├── ИмяПроекта.Tests.csproj                                 
        └── UnitTests.cs

Настройки проекта по умолчанию:

  • Code
    <Project Sdk="Microsoft.NET.Sdk">                                                                                
     <PropertyGroup>                                    // Определяет свойства проекта                              
      <OutputType>Exe</OutputType>               // Определяет выходной тип проекта                        
      <TargetFramework>net10.0</TargetFramework> // Определяет применяемую для компиляции версию фреймворка
      <ImplicitUsings>enable</ImplicitUsings>    // Определяет неявные пространства имен(без using в коде) 
     <Nullable>enable</Nullable>                        // Определяет ссылочные типы, допускающие значение null
     </PropertyGroup>                                                                                              
    </Project>
